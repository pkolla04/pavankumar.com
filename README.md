# pavankumar.com

Personal website and research portfolio built with [Hugo](https://gohugo.io/) and the [hugo-profile](https://github.com/gurusabarish/hugo-profile) theme. Deployed via GitHub Pages.

## Features

- Dark-first design with electric cyan/mint gradient accents
- Animated particle network background on hero section
- JSON-LD structured data (Person, WebSite, ScholarlyArticle, BlogPosting) for Google Knowledge Panel
- Open Graph + Twitter Card meta tags for rich social sharing
- Dedicated /publications page with DOI links, abstracts, citation counts
- Custom PK monogram favicon
- Google Analytics ready
- Auto-generation script to create blog + publication pages from research pipeline

## Local Development

```bash
# Start dev server (http://localhost:1313)
hugo server --buildDrafts

# Build for production
hugo --gc --minify
```

## Adding Content

### Blog Post

```bash
hugo new blogs/my-post-title.md
```

Then edit `content/blogs/my-post-title.md` with your content.

### Publication

Create a directory under `content/publications/`:

```
content/publications/my-paper-title/index.md
```

With frontmatter:

```yaml
---
title: "Paper Title"
date: 2026-01-01
venue: "IEEE XYZ"
doi: "10.1109/xxxxx"
abstract: "Paper abstract..."
pdf: "/papers/paper.pdf"
code: "https://github.com/pavankumarkolla/repo"
citation_count: 5
tags: ["AI", "cloud"]
---
```

### Auto-Generate from Research Pipeline

From the research-automation project:

```bash
python scripts/gen_website_content.py papers/drafts/<paper_dir>/ --website-dir ~/pavankumar.com
```

This creates both a blog post and publication page as drafts. Review, edit, then set `draft: false` to publish.

### Project

Add to the `projects` section in `hugo.yaml`:

```yaml
- title: "Project Name"
  content: "Description"
  image: /images/projects/project.png
  badges:
    - "Python"
    - "AI"
  links:
    - icon: fab fa-github
      url: https://github.com/pavankumarkolla/repo
```

## Deployment

Pushes to `main` automatically deploy via GitHub Actions to GitHub Pages.

### First-Time Setup

1. Create GitHub repo: `pavankumarkolla/pavankumar.com`
2. Push this code to the repo
3. In repo Settings > Pages, set source to "GitHub Actions"
4. Configure custom domain `pavankumar.com` in repo Settings > Pages

### Custom Domain DNS

Add these DNS records at your domain registrar:

| Type  | Name | Value                          |
|-------|------|--------------------------------|
| A     | @    | 185.199.108.153                |

| CNAME | www  | pavankumarkolla.github.io      |

## Google Analytics Setup

1. Go to https://analytics.google.com
2. Create a new GA4 property for `pavankumar.com`
3. Copy your Measurement ID (`G-XXXXXXXXXX`)
4. Add it to `hugo.yaml` under `services.googleAnalytics.id`

## Google Search Console Setup

1. Go to https://search.google.com/search-console
2. Add property `pavankumar.com`
3. Verify via DNS TXT record
4. Submit sitemap: `https://pavankumar.com/sitemap.xml`

## SEO Features

- **JSON-LD Person schema**: Helps Google build a Knowledge Panel for your name
- **ScholarlyArticle schema**: Publications get rich search results
- **BlogPosting schema**: Blog posts get enhanced search listings
- **Open Graph tags**: Rich previews on LinkedIn, Facebook
- **Twitter Cards**: Rich previews on Twitter/X
- **Canonical URLs**: Prevents duplicate content issues
- **Sitemap + robots.txt**: Auto-generated by Hugo

## Customization

- **Theme colors**: Edit the `color` section in `hugo.yaml`
- **Custom CSS**: Edit `static/style.css`
- **Particle animation**: Edit `static/js/particles.js` (colors, count, speed)
- **Profile photo**: Replace `static/images/me.png`
- **Favicon**: Replace `static/fav.png` (or edit `static/fav.svg`)
- **Structured data**: Edit `layouts/partials/head/extensions.html`
